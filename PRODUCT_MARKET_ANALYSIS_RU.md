## 2) Рыночный бенчмарк: Airbnb, Booking, Авито, Суточно.ру

## 2.1 Срез по платформам

### Airbnb
Сильные стороны:
- Сильная «двусторонняя» доверительная модель (рейтинги, отзывы, верификации).
- Глобально отлаженные UX-паттерны карточки объекта, календаря и коммуникаций.
- Гибкая экономика комиссий (есть split/single fee сценарии, изменения для PMS-хостов).

Наблюдения для бенчмарка:
- В 2025–2026 Airbnb менял структуру комиссий для части хостов, подключенных через PMS; важно учитывать прозрачность цены для гостя и payout для хоста.

### Booking.com
Сильные стороны:
- Очень сильный search+ranking и конверсионная оптимизация воронки.
- Огромный массив пользовательского доверия (verified reviews, гибкие фильтры).
- Сильный B2B-партнерский контур для объектов размещения.

Наблюдения:
- Условия видимости/ранжирования и партнерские механики завязаны на параметры комиссии, участие в программах и операционные сигналы.

### Авито (российский контур)
Сильные стороны:
- Огромная аудитория в РФ и сильный входящий трафик по недвижимости.
- Привычная для массового пользователя механика объявлений и чатов.

Ограничения с точки зрения вашего будущего продукта:
- Часто пользовательский сценарий ближе к классифайду, чем к end-to-end транзакционному бронированию в стиле OTA.
- Качество карточек/актуальность могут сильно зависеть от дисциплины размещения.

### Суточно.ру (российский нишевой лидер краткосрока)
Сильные стороны:
- Явный фокус именно на краткосрочную аренду.
- Транзакционная модель бронирования и механики доверия/настроек для арендодателей.
- Адаптированные под РФ процессы и коммуникация.

Наблюдения:
- Позиционирует себя как ведущий сервис онлайн-бронирования жилья; у хостов есть детальные настройки бронирования, включая instant booking с условиями по рейтингу гостя.

## 2.2 Сравнение по ключевым продуктовым блокам

## Поиск и discoverability
- **Airbnb/Booking**: лидируют по персонализации, релевантности, гибким фильтрам.
- **Авито**: сильный входящий охват, но UX под задачи краткосрока неоднороден.
- **Суточно**: доменно релевантные фильтры под краткосрочный спрос в РФ.
- **Рекомендация для вас**: в MVP обязательно сделать быстрый поиск по районам Н. Новгорода + карте + датам + гостям + price bands.

## Доверие и безопасность
- **Airbnb/Booking**: «сквозное» доверие через verified review/правила/поддержку.
- **Авито**: сильный бренд, но риск «классифайд-эффекта» (не всегда транзакция внутри платформы).
- **Суточно**: баланс маркетплейса и транзакционного контура.
- **Рекомендация**: внедрить жесткий trust-layer: верификация хоста, SLA ответов, модерация, антифрод-флаги, публичные правила отмен.

## Бронирование и платежи
- **Airbnb/Booking**: вылизанная воронка оплаты/подтверждения/уведомлений.
- **Авито**: в части недвижимости многие сделки могут уходить во внешние каналы.
- **Суточно**: сильнее «зашито» онлайн-бронирование.
- **Рекомендация**: для VK Mini Apps критично минимизировать шаги до подтверждения; избегать «разрывов» UX при оплате.

## Поставщики (арендодатели)
- **Booking/Airbnb**: зрелые инструменты тарифов, календарей, ограничений, комиссий.
- **Суточно**: хорошие практики настроек объекта и условий бронирования.
- **Рекомендация**: сделать кабинет хоста с календарем доступности, правилами заезда/выезда, ценовыми правилами выходные/праздники.

## Локализация под РФ и Н. Новгород
- **Глобальные игроки** уступают нишевым локальным сервисам в локальных нюансах.
- **Ваш шанс**: точечная глубина для одного города (районы, транспортные узлы, сценарии «командировка/мероприятие/семья»).

## 4) Рекомендованный MVP (8–12 недель) — уже под event-space вертикаль

1. Каталог и карта (район, вместимость, формат события, цена: бесплатные/платные, ставка за час/день).
2. Карточка площадки:
   - фото/планировка;
   - инвентарь (столы, стулья, звук, проектор, кухня и т.д.);
   - правила (шум, время, депозит, уборка);
   - минимальный слот бронирования.
3. Гибкое бронирование:
   - почасовое;
   - посуточное;
   - комбинированное (например, 3 часа + доп.час);
   - подтверждение по заявке или instant booking.
4. Онлайн-бронирование и оплата:
   - предоплата/полная оплата;
   - бесплатная бронь (с подтверждением владельца);
   - депозит/штрафы по правилам.
5. Кабинет гостя (мои брони, отмена, чат, документы/чек-листы для мероприятия).
6. Кабинет хоста (календарь, гибкие тарифы, правила, blackout dates, инвентарь, SLA ответа).
7. Trust & Safety модуль (верификация, репорты, спорные кейсы, ограничения по пользователю/объекту).

## 5) Архитектурные ориентиры под ваш стек

## VK Mini Apps (frontend)
- SSR/edge-friendly подход, легкие страницы, агрессивное кеширование каталога.
- Событийная аналитика по воронке: search -> view -> contact -> book -> pay.

## Vercel (app/runtime)
- Edge middleware для гео/AB/бот-фильтра.
- Serverless routes для API-бэкэнда и webhooks оплаты.

## Supabase (data/auth/storage)
- PostgreSQL + RLS-политики по ролям guest/host/admin.
- Storage для медиа + signed URLs.
- Realtime для чатов/статусов брони.
- Очереди/cron для авто-отмен неоплаченных броней и nightly sync цен.

1. Быстрее локальный онбординг хоста, чем у крупных OTA.

## 9) Критичные корректировки продукта под вашу новую вводную (высокий приоритет)

### 9.1 Доменные сущности, которые нужно добавить в модель данных
1. `venue_type` (лофт, зал, переговорка, коворкинг, студия и т.д.).
2. `rental_mode` (hourly, daily, mixed).
3. `pricing_rules` (базовая ставка, минимальные часы, цена выходных, ночной коэффициент).
4. `inventory_items` + `inventory_pricing` (входит в стоимость / оплачивается отдельно).
5. `booking_purpose` (день рождения, мастер-класс, бизнес-встреча и т.д.).
6. `free_booking_policy` (условия бесплатного бронирования, лимиты, требование подтверждения).

### 9.2 Фильтры поиска (обязательно для PMF)
- Цена: бесплатные / до X ₽/час / до X ₽/день.
- Слот: «на 2 часа сегодня», «на вечер пятницы», «на целый день».
- Вместимость и формат мероприятия.
- Наличие инвентаря и сервисов на площадке.
- Возможность instant booking.

### 9.3 Правила и антифрод для бесплатных бронирований
1. Лимиты на число активных бесплатных броней на пользователя.
2. Рейтинг надежности арендатора и штрафы за no-show.
3. Механизм «soft deposit» или подтверждение через VK identity для снижения фрода.
4. Приоритетная модерация площадок с аномально высоким числом отмен.

### 9.4 Монетизация (проверить 3 модели)
1. Комиссия с платных броней (классическая).
2. Подписка/промо для хостов (поднятие в выдаче, бренд-профиль, витрина).
3. Транзакционная доп.монетизация через платные опции (инвентарь, услуги площадки).

### 9.5 План развития на ближайшие 3 спринта
- **Спринт 1:** data model + миграции + фильтры каталога + карточка площадки с инвентарем.
- **Спринт 2:** почасовое/посуточное бронирование + правила бесплатной брони + базовый антифрод.
- **Спринт 3:** платежный контур, споры, KPI-дашборд (conversion, no-show, cancellation reasons).
