## 3. Структура базы данных Supabase

### 📊 Основные сущности (уже реализовано):

#### Пользователи и профили:
| Таблица | Назначение | Связи |
|-----------|-------------|--------|
| `profiles` | Профили пользователей | `vk_user_id`, `role` (guest/host/admin) |
| `app_users` | View для админки | Ссылка на `profiles` |

#### Площадки (venues/listings):
| Таблица | Назначение | Связи |
|-----------|-------------|--------|
| `listings` | Основная таблица площадок | `host_id` → profiles, `status` (draft/pending/active/blocked) |
| `venues` | View для админки | Ссылка на `listings` |
| `pricing_rules` | Правила ценообразования | `venue_id` → listings, hourly/daily rates, multipliers |
| `inventory_items` | Инвентарь площадки | `venue_id` → listings, name, price, quantity |
| `venue_photos` | Фото площадок | `venue_id` → listings, sort_order |
| `free_booking_policy` | Правила бесплатной аренды | `venue_id` → listings |
| `availability` | Календарь доступности | `listing_id` → listings, date, is_available |
| `listing_metrics` | Метрики площадки | `listing_id` → listings, views, favorites |

#### Бронирование:
| Таблица | Назначение | Связи |
|-----------|-------------|--------|
| `bookings` | Бронирования | `venue_id` → listings, `guest_id` → profiles, статусы |
| `booking_inventory_usage` | Использование инвентаря | `booking_id` → bookings, `inventory_item_id` |
| `favorites` | Избранное | `user_id` → profiles, `listing_id` → listings |

#### Платежи и финансы:
| Таблица | Назначение | Связи |
|-----------|-------------|--------|
| `payments` | Платежи | `booking_id` → bookings, провайдер (vkpay) |
| `payouts` | Выплаты хостам | `host_id` → profiles |
| `chargebacks` | Возвраты | `payment_id` → payments |

#### Коммуникации и модерация:
| Таблица | Назначение | Связи |
|-----------|-------------|--------|
| `messages` | Сообщения в чате | `booking_id` → bookings, `sender_id`, `receiver_id` |
| `notifications` | Уведомления | `user_id` → profiles |
| `reviews` | Отзывы | `booking_id`, `listing_id`, `author_id` |
| `reports` | Жалобы | `reporter_id` → profiles |
| `disputes` | Споры | `report_id` → reports |

#### Системные таблицы:
| Таблица | Назначение |
|-----------|-------------|
| `event_outbox` | Очередь событий для асинхронных задач |
| `webhook_events` | Webhook события (для платежей) |
| `audit_logs` | Логи действий пользователей |
| `host_trial_usage` | Использование пробного периода хостами |
| `host_promotion_usage` | Использование продвижения |

payment_status: 'created', 'authorized', 'paid', 'refunded', 'failed'

---

### 🔒 Row Level Security (RLS):
Все таблицы защищены RLS-политиками:
- **Guests**: видят только active listings, свои бронирования, свои профили
- **Hosts**: управляют своими площадками, видят бронирования своих площадок
- **Admins**: полный доступ ко всем данным

## 4. Конкурентные преимущества (наши сильные стороны):

### ✅ Что мы делаем лучше:
1. **Гиперлокальность**: Фокус на одном городе (Н.Новгород) = глубина проработки
2. **Скорость**: Бронирование внутри VK-экосистемы (один клик)
3. **Прозрачность**: Финальная цена без сюрпризов (как в OTA)
4. **Сценарии мероприятий**: Командировки, семейные поездки, локации города
5. **Поддержка**: Внутри привычного канала VK

### 🎯 Ниши, где мы выигрываем:
- **Event-space вертикаль**: Площадки для мероприятий (не просто жилье)
- **Почасовая аренда**: Для мастер-классов, встреч, фотосессий
- **Бесплатные площадки**: С особым режимом модерации

---

## 5. Главные риски:

1. **Недостаток supply** (объектов) на старте
   - *Решение*: Агрессивный онбординг хостов, бесплатный пробный период

2. **Юнит-экономика** при платном трафике
   - *Решение*: Оптимизация воронки, ретаргетинг

3. **Антифрод и качество объявлений**
   - *Решение*: Модерация, верификация, рейтинги

4. **Юридические требования**
   - *Решение*: Оферта, ПДн, правила отмены, KYC процессы

---

## 6. KPI первого квартала:

1. **Supply**: 50+ активных площадок в Н.Новгороде
2. **Booking conversion**: >15% от просмотра до брони
3. **No-show rate**: <5%
4. **Repeat booking**: >30% возвращающихся гостей
